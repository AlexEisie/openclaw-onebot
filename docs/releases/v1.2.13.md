# v1.2.13

## Security Defaults

- Requires explicit `allowFrom` authorization before OneBot text commands can run from inbound QQ private or group messages.
- Makes OpenClaw CLI sync during installation opt-in through `ONEBOT_SYNC_OPENCLAW_CLI=1`.
- Disables group auto reactions by default so new installs do not emit group reactions until configured.

## Runtime Hardening

- Stages outbound media in private directories with randomized filenames and restrictive file permissions.
- Stages inbound images and OneBot file attachments into OpenClaw local media before passing them to the agent, including files from replied messages.
- Supports split OpenClaw/NapCat containers by documenting the required shared bind mount and `sharedDir` / `containerSharedDir` mapping.
- Removes legacy `clawdbot` and `moltbot` peer dependency auto-installs from the npm dependency surface.
- Keeps `openclaw` as an optional peer dependency for hosts that already provide the runtime.

## Documentation And ClawHub

- Updates README and skill guidance for ClawHub payload installation, command authorization, access tokens, shared media directories, split OpenClaw/NapCat container mounts, inbound media staging, and safer defaults.
- Carries optional peer dependency metadata into the staged ClawHub package manifest.
